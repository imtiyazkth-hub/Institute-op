Director

  • ✅ सब कुछ देख सकते हैं

  • ✅ Teachers manage करना
  • ✅ Salary track करना
  • ✅ Competition manage करना
  • ✅ Operator & Teacher बनाना
  • ✅ Delete permissions देना
  • ✅ Reports & Settings

Data Entry Operator

  • ✅ Students add/edit
  • ✅ Bulk Upload
  • ✅ Competition results add/edit/delete
  • ✅ Reports देखना (fees के बिना)
  • ❌ Fees नहीं देख सकता
  • ❌ Teacher salary नहीं

  • ❌ Settings नहीं
👩‍🏫

Teacher

  • ✅ Hostel students देखना
  • ✅ Competition results देखना

  • ❌ Students add/edit/delete नहीं
  • ❌ Fees नहीं दिखतीं
  • ❌ Bulk Upload नहीं
  • ❌ Settings, Salary नहीं

PART 3
Operator को Delete Permission देना
Director decide करता है — कौन से students Operator delete कर सकता है
ℹ️
By default: Operator कोई भी student delete नहीं कर सकता। Director जब चाहे specific class या roll number range के लिए permission दे सकता है।
1
Settings page खोलें
Director login → Sidebar → ⚙️ Settings
2
"🔐 Operator Delete Permissions" section ढूंढें
Page के नीचे यह section है। यहाँ दो तरह की permission दे सकते हैं:

📚 Class के हिसाब से

Operator एक पूरी class के students delete कर सकेगा

Class select करें:

🔢 Roll Number Range से

Operator specific roll numbers वाले students delete कर सकेगा

Roll number range:
Settings → Operator Delete Permissions
🔐 Current Permissions
📚 Class 10 ✕
🔢 Roll 1–50 ✕
✕ पर click करके permission कभी भी हटा सकते हैं
✅
Permission देने के बाद Operator के student table में उन्हीं students के सामने 🗑 Delete button दिखेगा जिनके लिए permission है।

Password Reset करना

ℹ️
अभी website से directly password reset नहीं होता। इसके लिए:
Option 1: Teacher/Operator को खुद login करके Settings → Change Password से बदलने दें।
Option 2: Firebase Console → Authentication → Users → उस user के "..." → Reset password email भेजें।

Minimum 6 characters → "🔒 Update Password" → Done!

PART 5
Firebase Console से Manual User बनाना
अगर website काम न करे तो backup method
⚠️
यह method सिर्फ तब use करें जब website से add करना work न कर रहा हो। Normal use में website से ही users बनाएं।
1
Firebase Console → Authentication → Add User
console.firebase.google.com → आपका project → Authentication → Users → Add user
2
Email & Password डालें → Add user → UID Copy करें
3
Firestore → users collection → Add document
Document ID = UID (जो copy किया)
// Teacher के लिए:
Document ID: xK9mP2qR8sT4vW6nY1z  ← Auth UID
Fields:
  name  → string → "Rahul Kumar"
  role  → string → "teacher"    ← lowercase!
  email → string → "rahul@school.com"

// Operator के लिए:
Document ID: aB3cD4eF5gH6iJ7kL8m
Fields:
  name  → string → "Imran Khan"
  role  → string → "operator"   ← lowercase!
  email → string → "imran@school.com"
❌
Common Mistake: role field में Teacher या Operator (capital letter) लिखना। यह हमेशा lowercase होना चाहिए: teacher / operator / director

User Setup Checklist
Click करके mark करें

Teacher Setup

  • Director login किया
  • All Teachers → Add Teacher खोला
  • Teacher का नाम, email, password भरा
  • Monthly salary set किया
  • Save किया — popup में credentials देखे
  • Teacher को email & password दिया
  • Teacher का login test किया — teacher.html खुला

Operator Setup

  • Settings → Add Operator खोला
  • Operator का नाम, email, password भरा
  • Save किया
  • Operator को credentials दिया
  • Operator का login test किया — operator.html खुला

Operator Delete Permissions (Optional)

  • Settings → Operator Delete Permissions खोला
  • Class या Roll range select करके permission दी
  • Operator login में check किया — सही students पर Delete button दिखा

TROUBLESHOOT
Common Problems
ProblemFix
Teacher login करने पर index.html पर वापस आ जाता है Firestore → users collection → Teacher के document में role: "teacher" (lowercase) check करें। Document ID = Firebase Auth UID होनी चाहिए।
Operator को fees दिख रही हैं Check करें कि वो operator.html पर है न कि director.html पर। Firestore में role "operator" है?
Teacher को Delete button दिख रहा है Files ठीक से upload हैं? GitHub पर latest teacher.html है? Cache clear करें।
Operator को Delete button नहीं दिख रहा Settings → Operator Delete Permissions में class/roll permission दी है? Permission save हुई?
"Email ya password galat hai" error Firebase Console → Authentication → Users में वो email है? Password correct है?
New teacher को कुछ pages error दे रहे हैं Firestore Indexes बने हैं? (fees, salaries, compResults) — Console में error link से बनाएं।
